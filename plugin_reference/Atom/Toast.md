# Toast Overlay

JSON 타입: `toast`
시스템 상태 변화를 사용자에게 알리는 일시적 피드백 메시지. 화면 중앙 상단에 표시.

## 언제 쓰는가
- 액션이나 수정으로 시스템 상태가 변했을 때 (저장 완료, 삭제 완료 등)
- 사용자의 작업 흐름을 방해하지 않는 가벼운 피드백
- sections 최상위에 lnb/main과 같은 레벨로 추가

## 구성 요소
- **A. Container**: min-width 120px, max-width 480px, border-radius 8px, 어두운 배경(#2F3133)
- **B. Text**: 흰색 텍스트, 최대 3줄 (1줄 권장). max-width 초과 시 줄바꿈

## 조합 예시
- 삭제 후 `toast(message="삭제가 완료되었습니다")` 표시
- 저장 후 `toast(message="변경사항이 저장되었습니다")` 표시
- 화면 상단에 자동 배치 (중앙 정렬, 상단 40px 여백)

## Props
| prop | 설명 |
|------|------|
| message | 피드백 메시지 텍스트 |

## 예시
```json
{ "type": "toast", "message": "삭제가 완료되었습니다." }
{ "type": "toast", "message": "변경사항이 저장되었습니다." }
```

## 주의사항
- 화면 중앙 상단에 배치 (데스크톱 기준 상단 40px 여백)
- 1줄 이내의 짧은 메시지 권장
- 런타임에서는 1.5초 후 자동 사라짐 (Figma 정적 디자인에서는 표시 상태로 배치)
- Modal과 달리 사용자 액션 없이 자동으로 사라지는 피드백용
