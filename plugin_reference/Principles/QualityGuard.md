# Quality Guard — AI 생성 디자인 품질 가드레일

## 목적
Claude가 화면을 생성할 때 빠지기 쉬운 반복 패턴과 품질 문제를 사전에 방지한다.

---

## 안티패턴: 하지 말 것

### 레이아웃
- **매번 같은 구조 반복 금지**: 모든 화면을 Top + SearchBar + Table로 만들지 말 것. 화면 목적에 따라 구성을 변경할 것.
  - 목록 조회 → Top + 필터 + Table (적합)
  - 상세 페이지 → Top + Accordion 섹션들 (적합)
  - 대시보드/요약 → Card 그리드 + 핵심 지표 (적합)
  - 설정 → FormGroup 기반 폼 (적합)
- **동일한 카드 반복 금지**: 같은 크기, 같은 구조의 카드를 3개 이상 나열하지 말 것. 정보 유형에 따라 카드 구성을 차별화할 것.
- **의미 없는 장식 금지**: 기능 없는 아이콘, 빈 카드, 장식용 구분선 추가하지 말 것.

### 버튼
- **Primary 버튼 남용 금지**: 화면당 Primary 버튼 최대 1개. 나머지는 반드시 Secondary/Tertiary.
- **버튼 과다 금지**: 한 화면(Top 영역 기준)에 버튼 4개 이상 배치하지 말 것. 넘으면 우선순위를 정해 줄일 것.
- **Destructive를 Primary로 쓰지 말 것**: 삭제 전용 화면이 아닌 한, Destructive 버튼은 Secondary/Tertiary로.

### 테이블
- **컬럼 과다 금지**: 테이블 컬럼 8개 이상이면 사용자가 파악하기 어려움. 핵심 컬럼만 노출하고 나머지는 상세 페이지로.
- **모든 컬럼 동일 너비 금지**: 컬럼 내용에 따라 너비를 차별화할 것 (ID는 좁게, 이름/설명은 넓게).
- **빈 테이블 방치 금지**: rowCount가 0이면 반드시 빈 상태 메시지를 고려할 것.

### 정보 구조
- **한 화면에 모든 정보 나열 금지**: 섹션이 5개 이상이면 Accordion이나 Tab으로 그룹핑할 것.
- **같은 정보 반복 금지**: Top의 title과 Breadcrumb의 마지막 항목이 중복되지 않도록 주의.

## 품질 체크: 해야 할 것

### 시각 계층
- 가장 중요한 액션(Primary 버튼)이 시각적으로 가장 눈에 띄는가?
- 제목 → 설명 → 컨텐츠 → 액션 순서로 자연스럽게 읽히는가?
- 상태 정보(StatusBadge, Signal)가 한눈에 구분되는가?

### 일관성
- 같은 유형의 화면은 같은 구조를 따르는가? (목록 화면끼리, 상세 화면끼리)
- 같은 액션은 같은 버튼 variant를 사용하는가? (저장=Primary, 취소=Secondary)
- LNB 메뉴 활성 상태가 현재 화면과 일치하는가?

### 실용성
- 사용자가 이 화면에서 해야 할 일이 즉시 파악되는가?
- 다음 단계로 가는 경로가 명확한가?
- 에러/빈 상태가 고려되었는가?
