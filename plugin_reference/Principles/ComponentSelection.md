# 컴포넌트 선택 원칙

## 필터 선택 기준
| 상황 | 사용 컴포넌트 |
|------|-------------|
| 선택지가 한눈에 보여야 하고 항상 노출 | chipgroup |
| 선택지가 많거나 접어두는 게 나을 때 | filterbar (드롭다운) |
| 텍스트 직접 입력 검색 | searchbar |

## 탭 vs 칩그룹
- tab: 콘텐츠 전체를 전환 (테이블 위에서 "전체/처리중/완료" 필터)
- chipgroup: 다중 선택 가능한 태그 필터, 또는 카테고리 버튼

## 버튼 위계
- primary: 주요 CTA 1개 (송금 신청, 저장, 확인)
- secondary: 취소, 보조 액션
- tertiary / tertiary2: 낮은 우선순위 (내보내기, 더보기)
- 삭제/초기화는 primary 사용 금지

## 테이블 컬럼 타입
- text: 일반 텍스트 (기본값)
- badge: 상태값, enum (완료/처리중/실패)
- button: 행 단위 액션 (상세보기, 다운로드)
