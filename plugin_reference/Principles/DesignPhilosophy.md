# Design Philosophy

이 디자인 시스템이 따르는 핵심 원칙.

---

## 1. 단순함 = 일관성과 예측 가능성

> 기능을 줄이는 게 아니라, 언제나 같은 방식으로 동작하게 만드는 것.

- 같은 종류의 액션은 항상 같은 컴포넌트, 같은 위치, 같은 스타일을 쓴다
- 사용자가 "이건 어떻게 하지?"를 고민하지 않도록, 유사한 화면은 유사한 패턴을 따른다
- 컴포넌트 prop 이름과 동작 방식은 시스템 전반에서 일관되게 유지한다 (`size`, `variant`, `disabled` 등)

**적용 예시**
- 목록 페이지: 항상 `Top + SearchBar/Tab + Table` 구조
- 상세 페이지: 항상 `Top + Accordion 섹션들` 구조
- 버튼 위계: ButtonHierarchy 원칙에 따라 어떤 화면이든 동일하게 적용

---

## 2. 복잡도는 컴포넌트가 흡수한다

> 사용자(개발자, 기획자)가 몰라도 되는 복잡함은 컴포넌트 내부로 숨긴다.

- Controlled/Uncontrolled 패턴: `opened` prop이 없으면 컴포넌트가 자체 상태를 관리
- Zebra stripe, 행 색상, 간격 같은 시각적 규칙은 컴포넌트 내부에서 처리 — 사용하는 쪽에서 신경 쓰지 않는다
- 복잡한 레이아웃 계산(fill, shrink, overflow 등)은 컴포넌트가 책임진다

**적용 예시**
- `MdsAccordion.ListItem`: odd/even 배경색은 자동 적용 — 사용처에서 클래스 지정 불필요
- `MdsTable`: 컬럼 너비, 헤더 스타일, hover 상태를 내부에서 일관되게 처리

---

## 3. 점진적 공개 (Progressive Disclosure)

> 처음에는 핵심만, 필요할 때 깊은 정보를 제공한다.

- 목록 → 상세로 이어지는 정보 계층을 지킨다
- 요약 카드에는 핵심 지표만 노출, 세부 내용은 Accordion/Modal/상세 페이지로
- 선택적 기능은 기본 숨김 상태에서 사용자가 열어볼 수 있게 (Accordion defaultOpened, Tooltip 등)

**적용 예시**
- 대시보드: 요약 지표 카드 (숫자 강조)
- 상세 페이지: Accordion으로 섹션별 접기/펼치기
- 복잡한 폼: 고급 옵션은 접어두고 기본 흐름만 노출

---

## 4. 신뢰는 기능이 아니라 기본값

> 사용자가 시스템 상태를 항상 정확히 알 수 있어야 한다.

- 상태는 항상 명확하게 표시한다 (`StatusBadge`, loading spinner, error message)
- 에러 메시지는 "무엇이 잘못됐는지" + "어떻게 해결하는지"를 함께 제공한다
- 파괴적 액션(삭제, 초기화)은 반드시 확인 단계를 거친다 (Dialog)
- 비가역적 작업은 사용자가 결과를 예측할 수 있도록 명확한 경고를 노출한다

**적용 예시**
- 삭제 버튼 → Destructive 스타일 + 확인 Dialog 필수
- 폼 제출 중 → Primary 버튼 disabled + loading 표시
- 빈 상태 → 단순 공백이 아닌 안내 메시지 제공

---

## 5. 사용자에게는 단순한 경로, 전문가에게는 깊은 제어

> 초보자와 전문가가 같은 컴포넌트를 각자의 방식으로 쓸 수 있어야 한다.

- 기본 사용은 최소한의 prop으로 동작해야 한다
- 고급 커스터마이징은 추가 prop으로 옵트인 방식으로 제공한다
- 필수 prop은 최소화하고, 나머지는 합리적인 기본값을 가진다

**적용 예시**
- `MdsAccordion`: `section` prop만으로 기본 동작 / `opened` + `onChangeOpened`로 완전한 제어 가능
- `MdsTop`: `size`와 `title`만으로 충분 / `description`, `right`는 선택적 추가

---

## 핵심 요약

| 원칙 | 한 줄 요약 |
|------|-----------|
| 일관성 | 같은 상황엔 항상 같은 패턴 |
| 복잡도 흡수 | 어려운 건 컴포넌트가 처리 |
| 점진적 공개 | 처음엔 핵심만, 필요하면 더 |
| 신뢰 | 상태는 항상 명확하게 |
| 유연성 | 기본은 단순하게, 고급은 옵트인 |
