# Cognitive Load — 인지 부하 관리 규칙

## 목적
사용자가 화면을 보고 즉시 이해하고 행동할 수 있도록, 화면 구성 시 인지 부하를 관리한다.

---

## 8항목 체크리스트

화면을 생성할 때 아래 항목을 확인할 것. 2개 이상 실패하면 구조를 재검토.

| # | 항목 | 기준 |
|---|------|------|
| 1 | **단일 초점** | 화면에 하나의 주요 목적이 명확한가? |
| 2 | **청킹** | 정보가 소화 가능한 그룹(4개 이하)으로 나뉘어 있는가? |
| 3 | **그룹핑** | 관련 항목이 시각적으로 묶여 있는가? (Card, Accordion 등) |
| 4 | **시각 계층** | 가장 중요한 것이 가장 먼저 눈에 들어오는가? |
| 5 | **한 번에 하나** | 사용자가 한 가지 결정에 집중할 수 있는가? |
| 6 | **최소 선택지** | 동시에 보이는 선택지가 4개 이하인가? |
| 7 | **기억 부담 없음** | 이전 화면 정보를 기억하지 않아도 되는가? |
| 8 | **점진적 공개** | 복잡한 정보가 필요할 때만 드러나는가? (Accordion, Modal) |

---

## 실전 적용 규칙

### 선택지 수 관리
- **버튼**: 한 영역에 최대 3~4개. 그 이상이면 우선순위를 정해 줄일 것.
- **탭**: 최대 5개. 그 이상이면 카테고리 재구성.
- **필터**: 상시 노출 필터(ChipGroup)는 5개 이하. 많으면 FilterBar(드롭다운) 사용.
- **테이블 컬럼**: 핵심 컬럼 5~7개. 나머지는 상세 페이지로.

### 정보 계층 구성
- **1단계 (즉시 파악)**: 페이지 제목, 핵심 수치, Primary 액션
- **2단계 (스캔)**: 테이블/목록, 상태 배지, 필터
- **3단계 (필요시 확인)**: Accordion 내용, 부가 정보, 설정 옵션

### 점진적 공개 패턴
| 상황 | 패턴 |
|------|------|
| 부가 정보 숨기기 | Accordion (접기/펼치기) |
| 확인/경고 | Modal 또는 Dialog |
| 세부 데이터 | 목록 → 상세 페이지 이동 |
| 고급 필터 | FilterBar (드롭다운으로 숨김) |
| 대량 액션 확인 | Dialog (건수 + 경고 메시지) |

### 화면 유형별 적정 복잡도

| 화면 유형 | 적정 섹션 수 | 권장 구조 |
|-----------|-------------|----------|
| 목록 조회 | 2~3 | Top + 필터(1개) + Table |
| 상세 페이지 | 3~5 | Top + Accordion 2~4개 |
| 폼/입력 | 2~4 | Top + FormGroup 1~3개 |
| 대시보드 | 3~4 | Top + Card 2~3개 (내용 차별화) |
| 설정 | 2~3 | Top + FormGroup/Accordion |
